## Project Brief

With the rapid rise of AI capabilities, the company wanted technical and non-technical teams alike to start experimenting with GenAI tools across Waymo's stack.

To support in this effort as the labeling team, we wanted to work on:
1. Allowing users to easily auto-label Waymo run data using LLMs
2. Allowing users to self-label Waymo run data as ground truth for fine-tuning AI models

## The Solution

First, I mapped out the main user flows I wanted the final design to cover, and how the user would navigate through each step,
as shown in the diagram below. 

<figure>
<img src="/projects/easy-labeling-user-flow.png" alt="Image of user flow" class="hero"/>
</figure>

The main steps would be 1) Importing data (from other tools or from CSV, SQL, etc.), 2) Creating a project
in the Easy Labeling tool, and 3) Labeling data through self-labeling or auto-labeling, or sending labels out to other tools (eg. for fine tuning).

## Results

We chose to create an “Easy Labeling” tool in the existing ML internal application at Waymo. This would become a central hub for creating simple labeling requests. Bulk labeling (many thousands of examples), 3D bounding box labeling, and other complex labeling requests would not be in scope.

Here, users can create workspaces and upload datasets into experiments to be self-labeled or auto-labeled. An “Experiment” is a dataset + prompt + labeling method, for example an experiment could be: 1000 examples of puddles + “Will this splash someone?” + auto-labeled with AI. 

We also added options on existing tools to import data into “Easy Labeling” for seamless transition, for example from the “data searching” tool. Once the data was found, it could be easily imported into an Easy Labeling experiment.
