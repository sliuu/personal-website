## Project Brief

The UX Engineering team in Ads was building a new internal prototyping tool to allow UX designers and researchers to create functioning prototypes for testing new ad formats without requiring engineering hours. The first version of this tool was live before I joined, but it was clunky and technical, difficult to use, and had a lot of bugs. As a result, it was still only UX engineers (our team) using this tool to create prototypes for other teams. The technical bar was just too high.

## Discovery & Research

Learning to use the tool myself, I ran into many pain points. I didn't know where to start, I realized the tool expected some basic knowledge of HTML and CSS, and "next steps" were not obvious. Most of the tool just expected you to know what things meant already, and to have some deep technical knowledge.

I also spoke to some expected users, UX designers and researchers, who didn't seem to be convinced that this tool would be useful for them. They were used to having UX engineers build stuff for them, and had little interest in learning a new tool.

I gathered that this tool would have to be very simple for them to use, so simple that it would become easier to use than asking a UX engineer to build a prototype for them. I also gathered that the tool would have to be very obvious in its functionality, and not require a high technical background in HTML and CSS to use it. 

I also did a competitive analysis to get an idea of what patterns worked particularly well or did not. Looking at Framer and ProtoPie, I noticed both tools aggressively reduce cognitive load by exposing complexity progressively instead of immediately, for example.

## Approach

Modernize the UI to make the tool more enjoyable to use
Use clear imagery, icons, and tooltips to make the tool more obvious for new learners
Group functionality and settings to reduce cognitive load
Hide advanced settings
Provide templates and pre-made components so users aren't starting from scratch


## The Solution

For the final designs, I did an aesthetic overhaul to create a more enjoyable and modern experience using the tool, created more obvious groupings to not overwhelm the user with blocks of text, made interactions more obvious, and made different sections more distinct.

My goal was to make a simple flow, adding pre-made Youtube components to a board, adding simple interactions, and exporting a prototype, as easy as possible for a first-time user. I also wanted to make it possible for users to create more complex prototypes with custom components and variables, but not overwhelm the first-time user with this complexity.

Youtube components should surface first, rather than text fields and arbitrary boxes. Users should never need to start completely from scratch, especially in the Ads space. 

Interactions should look strikingly similar to Figma; we want the users to feel like they already know how to use the tool, we don't need to reinvent the wheel.
